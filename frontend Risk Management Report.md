# Anywhere Music 프론트엔드 기술 리스크 관리 보고서
# 1. 개요 (Overview)
* 본 문서는 Anywhere Music 크롬 확장 프로그램의 사용자 인터페이스(UI) 및 사용자 경험(UX) 개발 과정에서 직면한 주요 기술적 제약 사항과 이를 극복하기 위한 설계적 결정을 기술합니다.

## 2. 리스크 및 해결 과정
### 1. 팝업 폐쇄성으로 인한 상태 동기화 실패
> 발생 상황
>* 크롬 확장 프로그램의 팝업은 포커스를 잃는 즉시 소멸돼 버립니다. 따라서 다시 열 때마다 모든 DOM 상태가 초기화되어 현재 재생 중인 곡 정보나 진행 바가 0으로 돌아가는 문제가 발생했습니다.

> 해결
>* 상태 구독 및 가상 상태 비교 패턴.
>* 모든 재생 상태는 Background Service Worker가 소유하며, 팝업은 열리는 즉시 GET_CURRENT_STATE 액션을 보내 현재 상태를 획득합니다.
>* 불필요한 DOM 업데이트 방지: lastState 객체를 도입하여 백그라운드에서 온 데이터와 현재 UI 데이터를 비교하고, 변경된 부분만 선택적으로 렌더링하도록 최적화했습니다.
>* 영향: 불필요한 리소스 소모를 줄이고 팝업이 닫혔다 열려도 사용자에게 끊김 없는 경험을 제공합니다.


#### 2. 좁은 팝업 환경에서의 복잡한 인터랙션 구현
> 발생 상황
>* 380x600px이라는 제한된 공간 안에서 재생 목록 관리(삭제, 재생) 기능을 모두 넣기에는 버튼이 너무 많아 UI가 지저분해지는 문제가 발생했습니다.

> 해결
>* 슬라이드 액션(Slide Actions) UI 설계
>* 캡슐화된 인터랙션: 각 리스트 아이템을 inner와 actions 계층으로 분리하고, 클릭 시 왼쪽으로 140px 이동하며 버튼이 나타나는 슬라이드 방식을 채택했습니다.
>* 모바일형 UX: 마우스 클릭만으로 모바일 앱의 스와이프와 유사한 경험을 제공하여 시각적 간결함과 기능성을 동시에 확보했습니다.


#### 3. 다크 모드 및 Glassmorphism 시각적 피로도
> 발생 상황
>* 백그라운드 재생 특성상 어두운 환경에서 사용될 확률이 높으나, 너무 단조로운 검정 배경은 사용자에게 답답함을 줄 수 있다고 생각했습니다.

> 해결
>* 레이어 기반 디자인 시스템
>* Glassmorphism: backdrop-filter: blur(25px)를 사용하여 하단 네비게이션과 카드 UI에 입체감을 부여했습니다.
>* SVG 기반 아이콘: 외부 이미지를 쓰지 않고 config.js에 정의된 SVG Path 데이터를 직접 주입하여 로딩 속도를 높이고 고해상도 대응을 완료했습니다.
